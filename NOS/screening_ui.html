    System Architecture
    Five specialized LLM agents evaluate a Notice of Sale from different dimensions. Each runs in parallel with no inter-agent communication. Consensus is deterministic Python—not another LLM call.
    01
    Text Extraction
    NOS PDF → pdftotext -layout → raw text. Layout mode preserves maturity table alignment that other methods destroy.
    02
    LLM Structured Extraction
    Full text + 55-feature JSON schema → Claude API → validated structured JSON. Deterministic checks (par sum, deposit math, date logic) catch errors. Retry on failure.
    03
    Parallel Agent Screening
    Validated NOS JSON + Firm Profile → 5 independent API calls. Each agent sees only its relevant fields. No agent sees another's vote.

    Why multi-agent? A single LLM hedges. If 4 dimensions look great but one is a dealbreaker, a single model says "Conditional" with the problem buried in paragraph three. The MAS architecture enforces hard vetoes that a single LLM softens. Each agent has decomposed reasoning, different context, and the architecture mirrors how real desks work—different people with different data making different judgments.
