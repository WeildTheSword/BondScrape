ARCHITECTURE NOTES
------------------
- disco.js is self-contained IIFE: injects button into .topbar-spacer on
  DOMContentLoaded, manages its own AudioContext, canvas, and CSS injection
- Disco music uses separate AudioContext from main sounds.js to avoid
  interference; all nodes tracked in discoNodes[] for cleanup on stop
- Music loops via setTimeout re-scheduling: scheduleDiscoLoop() fires 0.5s
  before current 16-bar chunk ends, schedules next chunk seamlessly
- Reasoning path cards use same visual patterns as firm profile cards:
  rp-card, rp-metrics, rp-metric (mirrored naming convention with rp- prefix)
- Mode state stored in localStorage nos_settings: apiKey, liveMode, liveApi,
  _modeChosen — same keys as the old modal for backwards compatibility
