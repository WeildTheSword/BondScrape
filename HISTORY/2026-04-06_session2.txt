WHAT WAS DONE
-------------

1. NOS EXTRACTION PIPELINE (NOS/nos_extraction/)
   - schema.py: 55-feature JSON schema across 10 categories (issuer, bond ID,
     sale logistics, maturity, coupons, bid evaluation, redemption, delivery,
     credit, legal, bidder obligations). Includes field-to-agent mapping.
   - extract_text.py: pdftotext -layout extraction with pypdf fallback.
   - llm_extract.py: Claude/OpenAI structured extraction with retry on
     validation failure. Detailed per-field extraction guidance in system prompt.
   - validate.py: Deterministic checks (par sum, GFD math, date logic,
     maturity count, bond years, average maturity, required fields).
   - evaluate.py: Ground truth comparison harness with weighted field-level
     accuracy, maturity schedule validation, batch mode.

2. MULTI-AGENT SCREENING SYSTEM (NOS/nos_agents/)
   - agents.py: 5 screening agents (Sector Fit, Size & Capital, Structure,
     Distribution, Calendar). Parallel execution via ThreadPoolExecutor.
     Confidence calibration guidance in all system prompts.
   - consensus.py: 4-rule deterministic consensus (hard veto at 0.8+ Pass,
     unanimous interested, mixed → conditional, multiple low-confidence
     Pass → escalation with human review flag).

3. GROUND TRUTH (NOS/nos_test_set/ground_truth/)
   - Created validated ground truth JSON for all 10 test PDFs
   - All pass validation checks (par sum, field presence, date logic)
   - Extracted text files saved for all 10 PDFs via pdftotext -layout

4. FIRM PROFILES (NOS/firm_profiles/)
   - texas_regional.json: Lone Star Municipal Partners (TX, $25M max, retail)
   - northeast_institutional.json: Atlantic Capital Markets (NE, $100M max, institutional)
   - national_large.json: National Municipal Securities (24 states, $500M max)
   - small_boutique.json: Magnolia Capital Advisors (TN only, $10M max, 0 analysts)

5. PIPELINE RUNNERS & TOOLS
   - run_screening.py: End-to-end pipeline (PDF → extraction → validation →
     agents → consensus). Context-aware dry-run mode.
   - demo_compare.py: Firm comparison demo (single, 6-scenario, full 10×4 grid)
   - batch_extract.py: Batch extraction + evaluation across all test PDFs
   - generate_report.py: Formatted text screening reports
   - interactive_demo.py: Menu-driven interactive screening
   - run_tests.py: 33 self-tests covering all pipeline components
   - test_set_stats.py: Test set diversity statistics for presentations
   - run_full_demo.sh: One-command full demo sequence

6. WEB UI (NOS/*.html)
